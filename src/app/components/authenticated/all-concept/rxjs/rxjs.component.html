RxJS Documentation

RxJS (Reactive Extensions for JavaScript) is a library for reactive programming using Observables. It helps handle asynchronous operations like API calls, user interactions, and event handling in a clean and efficient way.

Think of RxJS as a way to handle data streams. A stream is like a sequence of ongoing events or values that can be processed in a structured way.

Observable

An Observable represents a data stream that emits values over time. It is responsible for producing data and notifying subscribers when new data is available, when an error occurs, or when the stream completes.

  • Role: Producer of data.
  • Functionality: Emits values, handles errors, and signals completion.
  • Methods
    • next(value): Emits a value to subscribers.
    • error(err): Sends an error notification.
    • complete(): Signals that no more values will be emitted.

Observer

An Observer listens to an Observable and reacts when it emits values. It is responsible for consuming data and handling the emitted values, errors, or completion signals.

  • Role: Consumer of data.
  • Functionality: Receives values, processes errors, and reacts when the stream completes.
  • Methods
    • next(value): Handles received values.
    • error(err): Handles errors in the stream.
    • complete(): Executes final actions when the stream ends.

RxJS Methods Documentation

1. of() - Create an Observable from values also example of .map()

Creates an observable from a list of values.

2. from() - Convert an array, promise, or iterable into an Observable

Converts arrays, promises, or iterables into an observable.

With a Promise:

Use Case: Convert arrays or promises into an observable stream.

3. interval() - Emit values at a fixed time interval

Emits sequential numbers every specified milliseconds.

Use Case: Useful for periodic polling or real-time updates.

4. timer() - Emit a value after a delay or at intervals

The timer() function can:

  • Emit a single value after a delay .
  • Emit multiple values at intervals.

For repeated execution:

Use Case: Delay execution or create a countdown timer.

5. filter() - Filter values

Filters emitted values based on a condition.

6. mergeMap() - Handle multiple Observables

Merges values from inner observables.

7. switchMap() - Cancel previous Observab

Switches to a new observable and cancels the previous one.

Use Case: Ideal for live search where only the latest request should be considered.
